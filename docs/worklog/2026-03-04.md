## Key Decisions

- Prioritized scaffolding and governance baseline before product feature modules.
- Added a single CI workflow with required jobs (`evidence-check`, `lint`, `build`) to align with branch protection intent.
- Selected Python standard-library contracts and `Protocol` interfaces for fast iteration without dependency overhead.
- Enforced auditability/explainability at the contract layer (not as optional behavior).
- Implemented a deterministic, end-to-end simulation slice before deeper module complexity.
- Persisted run artifacts with schema-versioned manifest for replayable analysis.
- Rebuild run index on each execution rather than incremental patching to avoid stale entries.
- Chose latest-two default comparison to reduce CLI friction during demos.
- Reporting should be generated from indexed artifacts to keep decision narratives evidence-backed.
- V1 dashboard should remain local-first against synthetic/indexed artifacts to avoid external dependency risk.
- Theme switching must happen through bucket tokens (no style-token hardcoding inside dashboard layout files).
- Updated `design-system/manifest.json` to reflect active outfit-based design-system state.
- Threshold tuning should evaluate precision/recall/F1 and vendor overflow together, not model quality alone.
- Positive labels for threshold evaluation are `phishing` and `brand abuse` to align with enforcement-focused actionability.
- Firecrawl is now the dashboard token authority; borrowed bucket contributions are restricted to structure/interaction only.
- Reframe dashboard IA to foreground program intent: intake -> investigation -> intel synthesis -> execution evidence.
- Split product into landing + operations + campaign-intelligence surfaces for clearer user journey and role context.
- Replace prior case lifecycle UI behavior with a new stage model optimized for operator decision flow.
